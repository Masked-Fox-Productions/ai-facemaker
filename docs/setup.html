AWS Setup Guide

Get your AWS credentials configured for Bedrock image generation in about 10 minutes.

Prerequisites

  • An AWS account (create one free)
  • A payment method on file (image generation costs ~$0.01-0.08 per image)

Step 1: Enable Bedrock Model Access

AWS requires you to explicitly enable each AI model before use.

  1. Go to the Amazon Bedrock Console
  2. Select a supported region from the top-right dropdown:
    • US East (N. Virginia) - us-east-1
    • US West (Oregon) - us-west-2
    • Europe (Frankfurt) - eu-central-1
  3. Click Model access in the left sidebar
  4. Click Modify model access
  5. Check the boxes for the models you want:
    • Amazon Titan Image Generator G1 - Fast, affordable (~$0.01/image)
    • Stability AI SDXL 1.0 - High quality (~$0.04/image)
    • Stability AI SD3.5 Large - Best quality (~$0.08/image)
  6. Click Next, then Submit
  7. Wait 1-2 minutes for access to be granted

Step 2: Create an IAM User

Create a dedicated user with limited permissions for the demo.

  1. Go to the IAM Users Console
  2. Click Create user
  3. Enter a username like bedrock-demo
  4. Click Next
  5. Select Attach policies directly
  6. Click Create policy (opens new tab)
  7. Select the JSON tab and paste: 
    {
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": [
                "bedrock:InvokeModel",
                "bedrock:ListFoundationModels"
            ],
            "Resource": "*"
        }
    ]
}
  8. Click Next
  9. Name it BedrockImageGeneration
  10. Click Create policy
  11. Return to the user creation tab and refresh the policy list
  12. Search for and select BedrockImageGeneration
  13. Click Next, then Create user

Step 3: Create Access Keys

  1. Click on your new user's name
  2. Go to the Security credentials tab
  3. Scroll to Access keys and click Create access key
  4. Select Other as the use case
  5. Click Next, then Create access key
  6. Important: Copy both values now:
    • Access key ID (starts with AKIA...)
    • Secret access key (only shown once!)
  7. Click Done

Step 4: Use the Demo

  1. Go to the ai-facemaker demo
  2. Select the same region where you enabled model access
  3. Paste your Access Key ID and Secret Access Key
  4. Click Connect to AWS
  5. Start generating portraits!

Security Tips

  • Never share your secret key - Treat it like a password
  • Use minimal permissions - The policy above only allows image generation
  • Rotate keys periodically - Delete and recreate keys every few months
  • Set up billing alerts - Go to AWS Budgets to set spending limits
  • Delete unused keys - Remove access keys you're no longer using

Troubleshooting

"Access Denied" error

  • Verify the IAM policy is attached to your user
  • Check you're using the same region where you enabled model access
  • Wait a few minutes after enabling model access

"Model not found" error

  • Not all models are available in all regions
  • Try US East (N. Virginia) for the widest model selection
  • Ensure the specific model is enabled in Model Access

"Invalid credentials" error

  • Double-check you copied the full access key and secret
  • Ensure there are no extra spaces
  • Verify the access key is still active in IAM console
